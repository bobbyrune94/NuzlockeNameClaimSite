Want Your Viewers and Community to Nickname Your Playthrough Team?

If you bring this discord bot into your server, you can let your community use commands to claim and nickname any Pokémon. People can make claims, view claims, edit their nicknames, change their claims, and remove claims through the bot using structured Discord commands.

Selecting a command will automatically bring up all the parameters to populate.

PokéNameBot Details

  • Claiming a Pokémon will also claim its associated evolution line
    • Exception: branched evolutions are split into their “primary” line (original evolution line) and “secondary” line (later additions)
      • Example: the Poliwag line is split into the primary line: [Poliwag, Poliwhirl, Poliwrath] and the secondary line [Politoed]
    • Exception: multi-branched evolutions are to be claimed independently
      • Example: Eevee and its evolutions are all split up. One person claims Eevee, another claims Vaporeon, another claims Jolteon, etc.
  • Regional Variants can be claimed separately.
    • Example: One person can claim Kantonian Vulpix while another can claim Alolan Vulpix
  • Users can nickname Pokémon based on their genders, but this will only work on Pokémon that have 2 genders. Use the “gendered” subcommand to do that (“/claim gendered …”)
    • Example: You may nickname the shinx line Tiggs for males and Tigress for females, but not a Porygon (genderless) or Nidoqueen (only female)
  • Once you claim a Pokémon, you cannot change that Pokémon for 3 months. You are still free to edit the nickname as much as you want to
  • Server admin will decide which roles are allowed to make permanent claims (ex. Mods and T3 subs can make permanent claims, everyone’s claim is permanent, etc.). All transient (non-permanent) claims will be wiped on Jan 1st every year.
  • There is a 15 second cooldown between commands to prevent command spam. The bot will send a follow up message once your cooldown is up.
    • Exception: you can call the /directions command as often as you want
  • The list of Pokémon available to claim should be up to date with the most recent games. If not, then the new Pokémon should be added soon (whenever Kenny has time to accumulate that data)

Special Notes to be Aware of:

  • Name Formatting: some pokemon with special characters or spaces in their names needed to be formatted differently.
    • Mime Family: Replace the space with a dash (-). I.e. mr-mime, mr-rime, mime-jr
    • Farfetch’d Line: Remove the apostrophe. I.e. farfetchd, sirfetchd
    • Regional Variants: add a dash (-), then the region they came from. Example: vulpix-alola, meowth-galar, qwilfish-hisui
  • Special Evolution Lines
    • Wurmple Line: since Wurmple evolves randomly into Silcoon or Cascoon, we are dividing it into 3 claimable lines
      • Wurmple
      • Silcoon/Beautifly
      • Cascoon/Dustox
    • Burmy Line: Burmy is a gender-based evolution, not a branched evolution like an initial look would tell you. Male burmy become Mothim while females become Wormadam. Thus, we’re treating it like a standard evolution line; claiming any one of those will give you all three.

Tech Breakdown

System Diagram

The PokéNameBot System is fairly simple. Broken down to its simplest components, it’s a database that stores data and a discord bot that views and modifies that data. Everything else around it is used to help with that functionality.

  1. The Website: This is what you’re looking at right now. The website was built to give users an easy way to view and search for existing and available claims.
  2. The Discord Bot: This is the primary interface for the system. Through the discord bot, you can make claims, edit existing claims, and view claim data for yourself and any Pokémon available.
  3. API Gateway: This is how public systems, such as the website and discord bot, can interface with the Databases. Not everyone should be able to manipulate the data, so this is our way of restricting access to it. API endpoints call Lambda Functions (described below) to perform any number of functions on database tables.
  4. Lambda Functions: This is how we actually manipulate and grab data from the databases. Depending on the functionality, the lambdas can edit tables to make/change a claim or get single entries or whole tables. I’ve also made lambdas to make onboarding new Pokémon and discord servers easier for my future sanity.
  5. Databases: This is where all the data is stored. Think of it like an excel spreadsheet that I can programmatically edit and get data from.

Evolution Lines Table

This table stores all the evolution lines and essential information for each pokémon. The most important data here is the evo-line field. Any time a claim comes in, it checks this table to see what other Pokémon are in that evolutionary line that will also get claimed.

Claims Tables

These tables store every single pokémon available to claim. Unclaimed pokémon will have UNDEFINED in the nickname while claimed pokémon (see Jolteon) will display the nickname and claimer’s discord username and ID. It also stores the next time the user can change their claim, as changing claims is restricted to once every 3 months.

This website takes the data from these tables and displays it in a more easily consumable and searchable format.
